## Top issues ranked by severity

1. **缺少“临场追问路线图”，长文可用但不够速记**
   - 文档内容很强，但 45 个问题对面试前复习偏长。
   - 建议增加一个“10 个必背问题 + 10 个加分追问 + 5 个高阶挑战题”的压缩版，帮助读者在 15 分钟内复盘。

2. **高级生产化问题还可以补强**
   - 已覆盖 trace、SLO、成本、灰度、回滚，但长期任务状态、异步队列、任务取消/暂停、版本化 eval、模型 fallback、rate limit、租户级配置等生产问题较分散。
   - 这些点是强技术型 PM 面试中容易拉开差距的追问。

3. **模型选择与模型生命周期讨论略浅**
   - 文档提到质量、延迟、成本、上下文长度和结构化输出，但没有形成一套“模型选型/路由/降级/版本变更”的回答模板。
   - 建议增加一题：如何在强模型、小模型、专用模型、开源/私有部署之间做产品取舍。

4. **Marketing / RevOps / Customer Support 迁移示例不足**
   - GTM/Sales 示例很好，但如果读者面试 AI Native PM 或 Agent PM，不一定只被问 Sales Agent。
   - 建议补充 2-3 个短例：Marketing campaign Agent、RevOps data cleanup Agent、Customer support escalation Agent，展示框架可迁移。

5. **来源部分可靠但需要更明确的“当前性”说明**
   - 文档写了资料核验日期，也列出大量官方链接，这是优点。
   - 但 OpenAI、LangChain/LangGraph、MCP 这类文档变化很快，建议在参考链接中注明访问日期/版本，并对 MCP spec 这类版本化链接说明“按当前规范版本引用”。

## Missing topics

- 模型选型与路由：强模型/小模型/开源模型/私有部署/模型 fallback 的 PM 判断框架。
- 生产化长任务：异步执行、任务取消、暂停恢复、checkpoint、后台通知、失败补偿。
- 版本治理：模型版本、prompt 版本、tool schema 版本、知识库版本与 eval 回归的关系。
- 数据治理责任：知识库 owner、更新 SLA、权限同步、过期资料下线、引用来源可信度。
- Agent 经济性：cost per successful task 之外，可补 ROI、seat-based vs usage-based、企业预算和限额策略。
- Build vs Buy：什么时候买 Agentforce / Copilot / HubSpot Breeze 类产品，什么时候自建垂直 Agent。
- 更高阶安全题：间接 prompt injection 的测试集、tool poisoning、MCP server 信任边界、第三方工具供应链审查。
- 非 Sales 场景短案例：Marketing、RevOps、客服、内部知识助手或运营自动化。

## Suggested edits

1. 在第 7 节前或第 13 节后新增“面试前 15 分钟速记版”：
   - 必背 10 题：Agent vs chatbot、RAG 失败、tool schema、memory、workflow vs agent、eval、成本、延迟、安全、MVP。
   - 加分 10 题：MCP、LangGraph、multi-agent、HITL、灰度、trace、prompt caching、模型路由、数据治理、业务价值。
   - 高阶 5 题：Build vs Buy、模型生命周期、长期任务状态、第三方工具风险、组织级 ROI。

2. 增加一题“如何做模型选型和模型路由”：
   - 回答应覆盖质量、延迟、成本、上下文长度、结构化输出、工具调用、隐私部署、fallback、A/B test、版本回归。

3. 增加一题“长任务 Agent 如何产品化”：
   - 覆盖异步执行、进度反馈、暂停/取消、checkpoint、重试、补偿、后台通知和审计。

4. 在 GTM 综合案例后补 3 个短迁移案例：
   - Marketing Agent：品牌 guardrail、渠道适配、A/B test、批量审核。
   - RevOps Agent：CRM 数据清洗、字段更新建议、审批和回滚。
   - Support Agent：知识检索、工单摘要、升级建议、用户隐私控制。

5. 在参考链接中补充版本/访问日期说明：
   - 尤其是 MCP specification、OpenAI API docs、LangGraph/LangChain docs。
   - 可把“资料核验日期：2026-06-04”扩展为“所有链接访问/核验日期”。

6. 增加一个“面试中不要这样答”的总表：
   - 例如“加 RAG 就不会幻觉”“Agent 越自主越好”“多 Agent 一定更高级”“上线后再做 eval”“用超级账号跑工具”。
