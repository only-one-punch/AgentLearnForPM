## Top issues ranked by severity

1. **“速查表”过长，临场复习效率不足**
   - 文档有 606 行，覆盖全面但信息密度很高。作为系统学习材料很好，作为面试前最后 10 分钟速查还不够锋利。
   - 影响：读者能学懂，但临场抓重点会有负担。
   - 建议：在开头或 Quick Memory Summary 后增加“一页式面试速查”：10 个最常问术语、5 个核心 tradeoff、5 个核心指标、5 个生产风险、3 句万能表达。

2. **少数术语边界需要更精确**
   - `Prompt Resource` 容易让人误以为 MCP prompts 是 resources 的一种；按 MCP 语义，tools、resources、prompts 是不同 server capabilities。建议改为 `Prompt / Prompt Template`，并说明 MCP prompt 与 resource 分属不同能力。
   - `Confidence Score` 的 PM 表达正确强调不能只信模型自报，但还可以补一句：生产决策通常用证据覆盖、检索质量、工具校验、历史表现和规则共同判断，而不是依赖一个模型分数。
   - `Vector Weakness` 建议拆得更清楚：多租户隔离、embedding 泄漏/反推、误召回、向量库权限绕过分别是不同风险。

3. **缺一个具体的运行 trace 示例**
   - GTM Agent 方案已经完整，但更像架构流程。建议增加一个 8-10 步迷你 trace：用户输入 -> 检索 query -> top evidence -> tool call -> generated outreach -> compliance review -> approval -> CRM writeback -> metrics logged。
   - 影响：读者在面试中会更容易把术语转成“系统如何跑起来”的故事。

4. **Source URL 稳定性可以再打磨**
   - 抽查发现 OpenAI `platform.openai.com/docs/guides/...` 链接会跳转到 `developers.openai.com/api/docs/guides/...`。虽然可访问，但知识库引用建议使用最终 URL。
   - MCP、OWASP、NIST 等关键链接可访问，整体 source quality 合格。

5. **术语优先级没有分层**
   - 当前所有术语在视觉上权重相近。对面试而言，`Agent / RAG / Tool Calling / Workflow / Eval / HITL / Prompt Injection / Cost per Task / Trace / Task Success Rate` 明显比部分扩展术语更高频。
   - 建议标注 `Must know / Good to know / Advanced`，帮助读者分配复习时间。

## Suggested edits

1. 在第 24 节后增加“一页式面试速查”：
   - 10 个必背术语：Agent、RAG、Tool Calling、Workflow、Memory、Eval、Trace、HITL、Prompt Injection、Cost per Task。
   - 5 个必讲 tradeoff：RAG vs fine-tuning vs long context、Agent vs workflow、copilot vs autopilot、single vs multi-agent、quality vs latency/cost。
   - 5 个必备指标：task success rate、faithfulness、retrieval recall、tool call accuracy、cost per successful task。

2. 在 GTM 示例中补一个小型 trace：
   - 输入：“帮我研究 Acme 是否适合触达。”
   - 检索：CRM + 新闻 + 官网 + call notes。
   - 生成：账户摘要 + buying signal + outreach angle。
   - 审核：证据、合规、品牌语气。
   - 执行：用户确认后写回 CRM 或草拟邮件。
   - 观测：trace、用户修改、回复率、会议转化、成本。

3. 修正 MCP 相关术语：
   - 将 `Prompt Resource` 改为 `Prompt / Prompt Template`。
   - 明确 MCP 中 tools、resources、prompts 是不同能力，prompt 不是 resource 的子类。

4. 将 OpenAI 参考链接更新为最终跳转后的 developers.openai.com URL：
   - `https://developers.openai.com/api/docs/guides/agents`
   - `https://developers.openai.com/api/docs/guides/agents-sdk`
   - `https://developers.openai.com/api/docs/guides/tools`
   - `https://developers.openai.com/api/docs/guides/structured-outputs`

5. 给术语增加优先级标记：
   - `Must know`：面试必问或生产必备。
   - `Good to know`：能提升技术可信度。
   - `Advanced`：用于加分或特定场景。
